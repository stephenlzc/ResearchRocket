# Coding Standards (Research)

- No fabricated outputs or mocked success metrics in production artifacts.
- Keep experiment code deterministic where possible.
- Use clear function-level docstrings for experiment entry points.
- Separate data preparation, training/inference, and evaluation stages.
- Emit structured logs for every run.

## Paper Standards

- Every main claim must reference either an experiment artifact or external citation.
- Tables/figures must be reproducible from stored outputs.
- Clearly label limitations and failure modes.
